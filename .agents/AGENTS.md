# Workspace Rules

## Local Model Delegation Mode
- **Core Instruction:** For all software engineering, code generation, refactoring, and code modification requests, the agent MUST delegate the actual code generation to the local model `qwen2.5-coder:14b` via the `ollama-local` MCP server (`ollama_chat` or `ollama_generate` tools).
- **Reasoning:** This minimizes the output tokens consumed from the Gemini API quotas.
- **Workflow:**
  1. The agent reads the workspace files and gathers context.
  2. The agent writes a prompt specifying the required code changes.
  3. The agent calls the local `ollama_chat` tool with the model `qwen2.5-coder:14b` to generate the code.
  4. The agent writes or modifies the files using the code generated by the local model.
